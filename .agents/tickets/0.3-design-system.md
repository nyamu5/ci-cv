# TICKET 0.3 — CI/CV Design System & Terminal Theme

**Stage:** 0 — run this before any UI work in Stage 1+
**Depends on:** Ticket 0.1 and 0.2 complete

**Goal:** Apply the CI/CV green terminal aesthetic globally via shadcn/ui CSS
variable override. Every shadcn component inherits the theme automatically.
No per-component colour overrides needed after this ticket.

---

## Context

CI/CV uses a refined green phosphor terminal aesthetic — JetBrains Mono
throughout, near-black background, #2ECC71 green as the primary colour,
zero border-radius everywhere. The design is intentional: the CI/CD pipeline
metaphor made visual.

Key rules that apply to every component built after this ticket:

- Zero rounded corners on everything (except the three macOS dots in TerminalWindow)
- Monospace font everywhere — no system-ui or sans-serif
- ASCII progress bars (████░░) instead of shadcn Progress on results pages
- Three accent colours only: green (primary), red (errors/flags), amber (warnings)
- No gradients, no shadows, no glow effects

### 5. Create lib/ascii-bar.ts

Used on every results page instead of shadcn Progress:

```typescript
/**
 * Renders a score as an ASCII block progress bar.
 * Example: asciiBar(74, 20) → '██████████████░░░░░░'
 */
export function asciiBar(score: number, width = 20): string {
  const clamped = Math.max(0, Math.min(100, score));
  const filled = Math.round((clamped / 100) * width);
  return "█".repeat(filled) + "░".repeat(width - filled);
}

/**
 * Returns the CSS class for a score value.
 * Use with the .score-high / .score-mid / .score-low utility classes.
 */
export function scoreClass(score: number): string {
  if (score >= 75) return "score-high";
  if (score >= 50) return "score-mid";
  return "score-low";
}

/**
 * Formats a job ID from a UUID for display.
 * Example: 'job_7f3a2b' from '7f3a2b91-...'
 */
export function formatJobId(uuid: string): string {
  return `job_${uuid.slice(0, 6)}`;
}
```

### 6. Create components/ui/pipeline-line.tsx

The `[1/4] analyser ████████ PASS 12.4s` row used in the running screen
and in the hero terminal. Reusable across both:

```tsx
interface PipelineLineProps {
  step: number;
  total: number;
  name: string;
  status: "waiting" | "running" | "pass" | "fail";
  elapsed?: string;
  progress?: number; /* 0–100, used when status is 'running' */
}

export function PipelineLine({
  step,
  total,
  name,
  status,
  elapsed,
  progress = 0,
}: PipelineLineProps) {
  const bar =
    status === "running"
      ? "█".repeat(Math.round((progress / 100) * 16)) +
        "░".repeat(16 - Math.round((progress / 100) * 16))
      : status === "pass"
        ? "████████████████"
        : status === "fail"
          ? "░░░░░░░░░░░░░░░░"
          : "░░░░░░░░░░░░░░░░";

  const statusLabel =
    status === "pass"
      ? "PASS"
      : status === "fail"
        ? "FAIL"
        : status === "running"
          ? "wait"
          : "    ";

  const barColor =
    status === "pass"
      ? "var(--g)"
      : status === "fail"
        ? "var(--rd)"
        : "var(--gd)";

  return (
    <div className="flex items-center gap-2 font-mono text-[10px] leading-7">
      <span style={{ color: "var(--gd)", width: "28px", flexShrink: 0 }}>
        [{step}/{total}]
      </span>
      <span style={{ color: "var(--gb)", width: "72px", flexShrink: 0 }}>
        {name.padEnd(10)}
      </span>
      <span style={{ color: barColor, flex: 1, letterSpacing: "-0.5px" }}>
        {bar}
      </span>
      <span style={{ color: barColor, width: "32px", flexShrink: 0 }}>
        {statusLabel}
      </span>
      {elapsed && (
        <span style={{ color: "var(--gd)", width: "36px", textAlign: "right" }}>
          {elapsed}
        </span>
      )}
    </div>
  );
}
```

### 7. shadcn component mapping (reference for all future tickets)

| shadcn component | CI/CV usage                                                     |
| ---------------- | --------------------------------------------------------------- |
| `Card`           | Terminal panel — dark bg, ghost border, zero radius             |
| `Badge`          | `[PASS]` `[FAIL]` `[HIGH]` `[MED]` labels                       |
| `Alert`          | Red flag block — use `destructive` variant                      |
| `Tabs`           | CV input tab switcher (paste / upload / jd)                     |
| `Toast`          | Pipeline status notifications                                   |
| `Textarea`       | Wrapped in a `>` prefix div — see PromptInput pattern           |
| `Separator`      | Thin ghost line between sections                                |
| `Skeleton`       | Pipeline loading placeholder                                    |
| `Progress`       | **Only** for the indeterminate running animation — styled green |

**Never use:** shadcn `Progress` for section score bars — use `asciiBar()` instead.
