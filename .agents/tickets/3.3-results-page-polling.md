# TICKET 3.3 — Results Page with Polling

**Stage:** 3 — User Interface
**Goal:** The /analysis/[id] page that handles all three states.

```
Build the results page and polling:

1. components/analysis-pending.tsx — client component:
   - Polls GET /api/analysis/[id] every 2 seconds
   - Animated progress bar + messages rotating every 3 seconds
     ("Checking for buzzword overload...", "Counting how many times you
     said 'passionate'...", "Assessing impact language...", "Calculating
     ATS keyword gaps...")
   - On status 'complete': swap in <AnalysisResult /> without a reload
   - On status 'failed': show the failure reason + a "Try again" link

2. app/analysis/[id]/page.tsx — server component:
   - Fetch the row by id; notFound() if missing
   - status pending/processing → render <AnalysisPending id={id} />
   - status complete → render <AnalysisResult result={...} />
   - status failed → render the failure state
   - generateMetadata: title "CV Analysis — Score: {n}/100", description
     from the summary

3. app/dashboard/page.tsx — server component, auth-gated:
   - Redirect to /login if no session
   - List the user's analyses (newest first): date, target role, overall
     score, status badge, link to the full analysis
```

**Verification:**

```
- pnpm typecheck passes
- pnpm build succeeds
- Submitting a CV shows the pending animation, then auto-swaps to results
- A shared /analysis/{id} URL opens correctly in an incognito window
- /dashboard lists past analyses for a logged-in user
```
