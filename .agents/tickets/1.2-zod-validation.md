# TICKET 1.2 — Zod Validation Schemas + Tests

**Stage:** 1 — Foundation (Data + Auth)
**Goal:** All validation schemas defined and unit tested.

```
Build the validation layer for cv-roaster:

1. In lib/validations.ts, define and export:

   a) SectionSchema — score (0-100), feedback, highlights[], improvements[]

   b) GatekeeperSchema — pass (boolean), rejection_reason (nullable),
      metadata (nullable object: candidate_name, primary_role,
      years_experience, top_skills[max 5]), injection_detected (boolean)

   c) AnalysisSchema — overall_score (0-100), summary, sections (5 named
      SectionSchemas), jd_match (nullable object: match_score,
      matched_keywords[], missing_keywords[], missing_skills[], fit_summary),
      red_flags[] (issue, severity enum, fix), rewrites[] (original,
      suggestions[length 2], reason)

   d) AnalyseRequestSchema — cv_text (string, 200-15000 chars),
      jd_text (optional string, max 5000), target_role (optional, max 200)

   Export inferred TypeScript types for each schema.

2. In tests/validations.test.ts, write Vitest tests:
   - AnalyseRequestSchema rejects cv_text under 200 chars
   - AnalyseRequestSchema rejects cv_text over 15000 chars
   - AnalyseRequestSchema accepts a valid request with optional fields omitted
   - AnalysisSchema rejects a section score above 100
   - AnalysisSchema accepts jd_match as null
   - GatekeeperSchema requires injection_detected to be present
   - AnalysisSchema rejects a rewrites entry with 1 suggestion (must be 2)
```
