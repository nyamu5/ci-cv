# TICKET 3.2 — Results Display Components

**Stage:** 3 — User Interface
**Goal:** All the components that render a completed analysis.

```
Build the analysis display components:

1. components/score-card.tsx — section score card: name heading, large
   colour-coded score (red <50, amber 50-74, green 75+), shadcn Progress
   bar, feedback text, green highlights list, amber improvements list.

2. components/jd-match-card.tsx — rendered only when jd_match is present:
   large match score, matched keywords as green badges, missing keywords
   as red badges, missing skills as amber badges, fit summary paragraph.

3. components/red-flags-list.tsx — flags sorted high → medium → low,
   each with a severity-coloured badge, issue text, and fix text.

4. components/rewrite-suggestions.tsx — per rewrite: original bullet in
   muted strikethrough, reason, two suggestions in green boxes each with
   a Copy button (toast on copy).

5. components/share-button.tsx — Copy link button, copies window URL,
   shows a "Copied!" toast.

6. components/analysis-result.tsx — composes the above: overall score +
   summary + ShareButton at top, a metadata strip ("Analysed as: {role}
   · ~{years} years"), optional JdMatchCard, a responsive grid of 5
   ScoreCards, RedFlagsList, RewriteSuggestions.
```

**Verification:**

```
- pnpm typecheck passes
- pnpm build succeeds
- Components render correctly when passed a mock AnalysisResult object
  (create a temporary mock to eyeball them)
```

**Commit:**

```
git add -A && git commit -m "feat: analysis results display components"
```
