# TICKET 4.3 — Deploy to Railway

**Stage:** 4 — Hardening & Deployment
**Goal:** Live, working production deployment.

```
Deploy cv-roaster to Railway:

1. Push the repo to GitHub (github.com/nyamu5/cv-roaster):
   - Create the repo on GitHub first
   - git remote add origin git@github.com:nyamu5/cv-roaster.git
   - git push -u origin main

2. Pre-deploy checks:
   - pnpm build passes locally with zero errors
   - pnpm typecheck passes
   - pnpm test passes
   - next.config.ts allows image domains: avatars.githubusercontent.com,
     lh3.googleusercontent.com

3. Railway setup:
   - Create a new Railway project from the GitHub repo
   - Railway auto-detects Next.js via nixpacks (no Dockerfile needed)
   - Add ALL environment variables from .env.example in Railway settings
   - Set the start command if needed: pnpm start
   - Deploy

4. Post-deploy configuration:
   - Update Supabase OAuth redirect URLs to the live Railway domain
   - Deploy the Trigger.dev tasks to production:
     pnpm dlx trigger.dev@latest deploy
   - Confirm TRIGGER_SECRET_KEY in Railway matches the prod environment

5. Smoke test the live site:
   - Anonymous analysis end to end
   - Sign in with GitHub
   - The analysis appears in the dashboard
   - A shared analysis URL shows the OG image
   - An injection-attempt CV fails gracefully
```

**Verification:**

```
- The live Railway URL loads
- Full anonymous flow works: submit → pending → complete
- OAuth sign-in works on the live domain
- Background job completes in production (check Trigger.dev dashboard)
- LangSmith shows production runs tagged by stage
- OG image renders when the URL is shared
```

**Commit:**

```
git add -A && git commit -m "chore: production deployment configuration"
git tag v1.0.0
git push origin main --tags
```
