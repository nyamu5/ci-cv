# TICKET 4.1 — Error Handling & Polish Pass

**Stage:** 4 — Hardening & Deployment
**Goal:** Graceful failure everywhere; consistent UX.

```
Harden the application:

1. Error states:
   - Rate limited: clear toast with the reset time
   - Analysis failed: results page shows reason + retry that re-triggers
     the job
   - PDF parse failure: inline error in the upload zone
   - OpenAI/AI validation error: the job marks 'failed' with a generic
     user-facing message (no raw error leakage)

2. Loading states: skeletons on the dashboard and results page initial load.

3. Add a global error.tsx and not-found.tsx in app/.

4. Accessibility quick pass: form inputs have labels, buttons have
   accessible names, colour is not the only signal on scores (add text).

5. Responsive check: form, score grid, and nav all work at 375px width.

6. Run pnpm check and fix all Biome warnings.
```

**Verification:**

```
- pnpm typecheck passes
- pnpm check passes with zero warnings
- pnpm build succeeds
- Manually trigger each error path and confirm a graceful message
- Layout is clean at mobile (375px) and desktop widths
```
