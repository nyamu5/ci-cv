# TICKET 2.1 — PDF Parsing with Garble Detection + Tests

**Stage:** 2 — Core Engine (PDF + Rate Limit + AI Pipeline)
**Goal:** PDF text extraction that flags low-quality output.

```
Build the PDF parsing layer:

1. lib/pdf.ts — export extractTextFromPDF(buffer: Buffer):
   - Use pdf-parse to extract text
   - Garble heuristic: compute alphanumeric-to-total character ratio and
     average word length. Flag qualityWarning = true if ratio < 0.6 OR
     average word length > 15.
   - Return { text: string, pageCount: number, qualityWarning: boolean }
   - Add a JSDoc block explaining WHY this exists (multi-column PDF bleed,
     and the cost reasoning for not using vision parsing).

2. app/api/upload/route.ts — POST handler:
   - Accept multipart/form-data, field "pdf"
   - Validate: application/pdf only, max 5MB
   - Call extractTextFromPDF
   - Return { text, pageCount, qualityWarning }
   - Typed error responses: 400 invalid file, 500 parse failure

3. tests/pdf.test.ts — Vitest tests for the garble heuristic:
   - A normal sentence string scores qualityWarning = false
   - A garbled string (random chars, no spaces) scores qualityWarning = true
   - An empty string is handled without throwing
   (Test the heuristic function directly — extract it as a pure exported
   function isLikelyGarbled(text: string) so it is unit-testable.)
```

**Verification:**

```
- pnpm typecheck passes

  confirm clean text returns
```
