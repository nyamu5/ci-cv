# TICKET 1.4 — Auth UI (Google + GitHub OAuth)

**Stage:** 1 — Foundation (Data + Auth)
**Goal:** Working sign-in flow with two OAuth providers.

```
Implement the auth UI for cv-roaster:

1. app/auth/callback/route.ts — GET handler that exchanges the OAuth code
   for a session (supabase.auth.exchangeCodeForSession) and redirects to
   /dashboard on success, /login?error=auth on failure.

2. components/auth-button.tsx — client component:
   - "Continue with Google" button → supabase.auth.signInWithOAuth({
     provider: 'google', options: { redirectTo: <origin>/auth/callback }})
   - "Continue with GitHub" button → same with provider: 'github'
   - Use shadcn Button components with provider icons

3. components/nav.tsx — server component:
   - Left: "CV Roaster 🔥" linking to /
   - Right: if session exists → user email + Sign out button (client
     sub-component calling supabase.auth.signOut())
   - Right: if no session → "Sign in" link to /login

4. app/login/page.tsx — centred card with heading "Sign in to save your
   analyses" and the AuthButton component.

5. Update app/layout.tsx — include <Nav /> and the shadcn <Toaster />.

NOTE: Google and GitHub OAuth credentials must be configured in the
Supabase dashboard (Authentication → Providers). Add a comment block in
auth-button.tsx documenting the required redirect URL:
{SUPABASE_URL}/auth/v1/callback
```

**Verification:**

```
- pnpm typecheck passes
- pnpm build succeeds
- After configuring providers in Supabase dashboard: clicking "Continue
  with GitHub" completes sign-in and lands on /dashboard
- Nav shows the user email when logged in, "Sign in" when logged out
```

git add -A && git commit -m "feat: Google and GitHub OAuth sign-in"
