# TICKET 2.4 — OpenAI Multi-Model Pipeline (LangSmith)

**Stage:** 2 — Core Engine (PDF + Rate Limit + AI Pipeline)
**Goal:** Two-stage AI functions, observable via LangSmith.

```
Build the AI pipeline in lib/openai.ts:

1. Install the LangSmith SDK:
   pnpm add langsmith

2. Initialise the OpenAI client and wrap it with LangSmith tracing:
   import OpenAI from "openai";
   import { wrapOpenAI } from "langsmith/wrappers";

   const openai = wrapOpenAI(new OpenAI({ apiKey: OPENAI_API_KEY }));

   Tracing is enabled via env vars (read by the langsmith SDK at runtime):
     LANGSMITH_TRACING=true
     LANGSMITH_API_KEY=...
     LANGSMITH_PROJECT=cv-roaster
   Do NOT set a custom baseURL — use the default OpenAI endpoint.

3. Export gatekeeperCheck(cvText: string):
   - model: gpt-4o-mini, temperature: 0, max_tokens: 300
   - response_format: json_object
   - Tag the call with LangSmith metadata { stage: "gatekeeper" }
     (pass via the langsmith run config / wrapped-client options)
   - System prompt: document classifier + injection detector + metadata
     extractor (use the Stage 1 prompt from the spec)
   - Validate against GatekeeperSchema
   - Return { result: GatekeeperResult, costUsd } using estimateCost

4. Export analyseCV(cvText, jdText, targetRole, metadata):
   - model: gpt-4o, temperature: 0.3, max_tokens: 2500
   - response_format: json_object
   - Tag the call with LangSmith metadata { stage: "analyser" }
   - System prompt: the hardened analyser prompt, enriched with the
     Stage 1 metadata line ("Candidate: X, N years, role Y")
   - Validate against AnalysisSchema
   - Return { result: AnalysisResult, costUsd } using estimateCost

5. Both functions: on Zod validation failure, throw a typed
   AIValidationError with the raw response attached for debugging.
```
