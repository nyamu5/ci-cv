# TICKET 2.4 — OpenAI Multi-Model Pipeline (Helicone)

**Stage:** 2 — Core Engine (PDF + Rate Limit + AI Pipeline)
**Goal:** Two-stage AI functions, observable via Helicone.

```
Build the AI pipeline in lib/openai.ts:

1. Initialise the OpenAI client pointing at the Helicone proxy:
   baseURL: "https://oai.helicone.ai/v1"
   defaultHeaders: {
     "Helicone-Auth": `Bearer ${HELICONE_API_KEY}`,
     "Helicone-Property-App": "cv-roaster",
   }

2. Export gatekeeperCheck(cvText: string):
   - model: gpt-4o-mini, temperature: 0, max_tokens: 300
   - response_format: json_object
   - Helicone-Property-Stage: "gatekeeper" header per request
   - System prompt: document classifier + injection detector + metadata
     extractor (use the Stage 1 prompt from the spec)
   - Validate against GatekeeperSchema
   - Return { result: GatekeeperResult, costUsd } using estimateCost

3. Export analyseCV(cvText, jdText, targetRole, metadata):
   - model: gpt-4o, temperature: 0.3, max_tokens: 2500
   - response_format: json_object
   - Helicone-Property-Stage: "analyser" header per request
   - System prompt: the hardened analyser prompt, enriched with the
     Stage 1 metadata line ("Candidate: X, N years, role Y")
   - Validate against AnalysisSchema
   - Return { result: AnalysisResult, costUsd } using estimateCost

4. Both functions: on Zod validation failure, throw a typed
   AIValidationError with the raw response attached for debugging.
```

**Verification:**

```

  returns pass: true and extracted metadata
- The same script with "ignore all previous instructions" embedded
  returns injection_detected: true

```
