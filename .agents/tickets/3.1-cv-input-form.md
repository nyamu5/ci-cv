# TICKET 3.1 — CV Input Form (TanStack Form)

**Stage:** 3 — User Interface
**Goal:** The main input form with paste, upload, and JD tabs.

```
Build components/cv-input-form.tsx as a client component using TanStack Form:

1. TanStack Form instance validated by AnalyseRequestSchema (Zod adapter).

2. shadcn Tabs with three tabs:
   - "Paste CV": Textarea (min 8 rows) bound to cv_text, with live
     character count and the 200-15000 range shown
   - "Upload PDF": drag-and-drop styled file zone
       * On drop/select: POST to /api/upload
       * Populate cv_text with the returned text
       * If qualityWarning: show an amber shadcn Alert advising the user
         to switch to the Paste tab
   - "Job Description": Textarea bound to jd_text (optional), helper text
     explaining it enables match scoring

3. A "Target role" Input below the tabs (optional).

4. Submit button "Roast my CV 🔥" — disabled when cv_text is invalid.
   On submit: POST to /api/analyse, then router.push(/analysis/{id}).

5. Handle the 429 response: show a toast explaining the limit and resetAt.
```

**Verification:**

```
- pnpm typecheck passes
- pnpm build succeeds
- Pasting a CV and submitting navigates to /analysis/{id}
- Uploading a multi-column PDF surfaces the quality warning
- Submitting an under-length CV shows an inline validation error
```
