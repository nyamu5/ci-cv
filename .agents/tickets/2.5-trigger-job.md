# TICKET 2.5 — Trigger.dev Background Job

**Stage:** 2 — Core Engine (PDF + Rate Limit + AI Pipeline)
**Goal:** The full analysis pipeline running as a background task.

```
Build the background job with Trigger.dev v3:

1. Initialise Trigger.dev:
   pnpm dlx trigger.dev@latest init
   (Configure with the project ref; TRIGGER_SECRET_KEY in .env.local)

2. trigger/run-analysis.ts — define a task "run-analysis":
   - Payload: { analysisId: string }
   - Step 1: fetch the analyses row by id
   - Step 2: update status → 'processing'
   - Step 3: call gatekeeperCheck(cv_text)
       * If pass is false OR injection_detected is true:
           update status → 'failed', store { error, reason } in result,
           return early
   - Step 4: call analyseCV(cv_text, jd_text, target_role, metadata)
   - Step 5: update the row → status 'complete', result, cost_usd
     (gatekeeper cost + analyser cost), completed_at = now()
   - Wrap in try/catch: on any error, set status → 'failed'
   - Configure retries: 3 attempts for transient failures

3. app/api/trigger/route.ts — the Trigger.dev route handler
   (per Trigger.dev v3 Next.js integration docs).
```

**Verification:**

```
- pnpm typecheck passes
- pnpm dlx trigger.dev@latest dev runs the local Trigger worker
- Manually inserting a 'pending' analyses row and triggering the task
  moves it pending → processing → complete within 30 seconds
- A CV with an injection attempt ends as 'failed' with a stored reason
```
