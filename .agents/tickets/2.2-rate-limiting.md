# TICKET 2.2 — Rate Limiting with Upstash Redis

**Stage:** 2 — Core Engine (PDF + Rate Limit + AI Pipeline)
**Goal:** Distributed rate limiting that survives restarts.

```
Implement rate limiting:

1. lib/rate-limit.ts:
   - Create an Upstash Redis client from UPSTASH_REDIS_REST_URL and TOKEN
   - Export checkRateLimit(identifier: string, isAuthenticated: boolean):
       * Sliding window via @upstash/ratelimit
       * Anonymous: 5 requests per 24h
       * Authenticated: 20 requests per 24h
       * Return { success, remaining, resetAt }
   - Export getIdentifier(req): builds "anon:{ip}" from x-forwarded-for /
     x-real-ip headers, or "user:{id}" if a user id is passed.

2. Confirm graceful behaviour: if Redis is unreachable, fail OPEN
   (allow the request) but log a warning — never block a user because
   the rate limiter is down.
```

**Verification:**

```
- pnpm typecheck passes
- Manual: call checkRateLimit in a temporary script 6 times with the same
  identifier — the 6th returns success: false
- Disabling the Redis env vars causes requests to fail open, not crash
```
