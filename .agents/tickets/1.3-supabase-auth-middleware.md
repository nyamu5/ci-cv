# TICKET 1.3 — Supabase Clients & Auth Middleware

**Stage:** 1 — Foundation (Data + Auth)
**Goal:** Browser + server Supabase clients and route protection.

```
Set up Supabase clients and auth middleware:

1. lib/supabase/client.ts — browser client via createBrowserClient
   from @supabase/ssr.

2. lib/supabase/server.ts — server client via createServerClient with
   Next.js cookie handling (next/headers cookies()). Export an async
   getSession() helper.

3. middleware.ts at project root:
   - Refresh the Supabase session on every request
   - If the request path starts with /dashboard and there is no session,
     redirect to /login
   - Matcher config to exclude static assets and _next

4. Verify TypeScript types resolve for both clients.
```

**Verification:**

```
- pnpm typecheck passes
- pnpm dev runs; visiting /dashboard while logged out redirects to /login
- No console errors related to Supabase client creation
```
