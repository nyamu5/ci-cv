# TICKET 1.1 — Database Schema & Drizzle Setup

**Stage:** 1 — Foundation (Data + Auth)
**Goal:** Type-safe database layer with the full schema.

```
Build the database layer for cv-roaster:

1. In lib/db/schema.ts, define with Drizzle:
   - A pgEnum "analysis_status": pending, processing, complete, failed
   - The analyses table:
       id            uuid PK default gen_random_uuid()
       user_id       uuid nullable
       cv_text       text not null
       jd_text       text nullable
       target_role   varchar(200) nullable
       status        analysis_status default 'pending'
       result        jsonb nullable
       cost_usd      numeric(10,6) nullable
       created_at    timestamp with timezone default now()
       completed_at  timestamp with timezone nullable
   - Export the inferred types: Analysis (select), NewAnalysis (insert)

2. In lib/db/index.ts:
   - Create a pg Pool using DATABASE_URL
   - Export the Drizzle db instance typed with the schema

3. Generate the migration:
   pnpm db:generate
   (This creates SQL in drizzle/ — review it looks correct)

4. Push to Supabase:
   pnpm db:push
```

**Verification:**

```
- pnpm typecheck passes
- pnpm db:studio opens and shows the analyses table with all columns
- The analysis_status enum exists in the database
```

**Commit:**

```
git add -A && git commit -m "feat: database schema and Drizzle setup"
```
